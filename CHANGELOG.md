# 更新记录

## 0.7.2

- Windows 插件、本地推理引擎与构建脚本以 MIT License 开源发布。
- 修复安装结束时隐藏真实引擎验证错误的问题。
- 安装界面现在会区分 Visual C++ 运行库、NVIDIA / CUDA 组件和权限拦截，并给出对应处理办法。
- 安装诊断同时保存到 `%LOCALAPPDATA%\EagleAutoTagger\logs\setup.log`，方便远程排查。
- 推理引擎事件解析可忽略第三方组件输出的非 JSON 信息，避免把正常诊断误判为安装失败。

## 0.7.1

- 将后续源码转入私有开发仓库，公开仓库只保留产品说明、安装包和问题反馈。
- 增加公开测试版使用条款，明确官方安装包的使用与再分发边界。
- 保留并补充第三方组件许可说明；0.7.0 及更早版本继续适用其原有 MIT License。

## 0.7.0

- 新增“导入本地模型”，支持从网盘取得模型后离线导入。
- 导入时校验固定文件大小和 SHA-256，通过后才原子切换缓存。
- Python 安装按 USTC 镜像、GitHub 官方源自动回退。
- GPU 依赖按 USTC、清华、PyPI 官方源自动回退，并复用下载缓存。
- 精简 GitHub 根目录：引擎源码只保留一份，开发脚本与文档分别归入 `scripts`、`docs`。

## 0.6.1

- 插件正式更名为 ds Eagle Tagger。
- 点击“开始新任务”后清空上一项任务的文件夹选择、搜索与统计结果。
- 自动检测 NVIDIA 显卡和可用显存，并推荐 Batch。
- 显存不足时自动按 8 → 4 → 2 → 1 降级，并按显卡记住安全值。
- 支持补充已有标签、本地处理记录、记录跨硬盘迁移及批准模型更新。
- 完善 Windows 一键安装包、下载进度、失败项目定位与公开版隐私说明。
